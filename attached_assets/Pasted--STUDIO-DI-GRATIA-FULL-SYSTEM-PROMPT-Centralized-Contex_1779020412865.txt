## ENTITY 3 — FILMCRAFT

**Status: Confirmed. Built. Functional.**

FilmCraft is a cinematic production operating system. A private, all-in-one workspace for running a film or documentary from first idea through distribution. 15 fully functional modules. One per-project workspace.

**What it is:** Structured record-keeping and planning system — the production bible, lived in daily.
**What it is not:** Script editor. Video editor. Live multi-user collaboration tool.
**Current center:** Ghost Theory EP1 "She Looked Back"
**Status:** Private pre-release. Small tester pool planned. Not yet public.
**Future:** Potential commercial product. Possible sale. No timeline confirmed.

### Tech Stack
| Concern | Technology |
|---|---|
| Monorepo | pnpm workspaces |
| Node.js | 24 |
| TypeScript | 5.9 |
| Frontend | React + Vite (`artifacts/filmcraft`) |
| Routing | Wouter |
| Server state | TanStack React Query |
| UI | Tailwind CSS + shadcn/ui |
| API | Express 5 (`artifacts/api-server`) |
| Database | PostgreSQL + Drizzle ORM |
| Validation | drizzle-zod (server), Zod v4 (shared) |
| API contract | OpenAPI 3 (`lib/api-spec/openapi.yaml`) |
| API codegen | Orval → React Query hooks + Zod schemas |
| Build | esbuild (CJS bundle for server) |

**Critical constraint:** Server routes must import Zod via `drizzle-zod` schemas (`insertXSchema` from `@workspace/db`), NOT via `import { z } from "zod/v4"` — esbuild cannot resolve the `/v4` subpath.

### Key Commands
```bash
pnpm run typecheck
pnpm run build
pnpm --filter @workspace/api-spec run codegen
pnpm --filter @workspace/db run push
```
Never run `pnpm dev` at workspace root. Never call service ports directly — use proxy at `localhost:80`.

### Architecture
```
artifacts/
  api-server/src/
    routes/         — Express route handlers (one file per domain)
    index.ts        — App bootstrap
  filmcraft/src/
    pages/projects/ — All 15 module pages
    components/
      layout.tsx    — Sidebar + mobile nav (PROJECT_NAV drives all links)
      ui/           — shadcn/ui primitives
    App.tsx         — Wouter route definitions

lib/
  api-spec/openapi.yaml          — Single source of truth for API contract
  api-client-react/src/generated/ — Orval-generated hooks (DO NOT edit manually)
  api-zod/src/generated/          — Orval-generated validators (DO NOT edit manually)
  db/src/schema/                  — Drizzle table definitions (one file per domain)
```

### Database Schema Files
| File | Contents |
|---|---|
| `projects.ts` | Film projects (title, logline, genre, format, director, producer, budget, status) |
| `beats.ts` | Beat sheets — Save the Cat 15-beat structure |
| `mindmap.ts` | Mind map nodes (position, title, body, edges as JSON) |
| `characters.ts` | Character bibles (name, age, description, backstory, motivation, arc, notes) |
| `scenes.ts` | Script scenes — sceneNumber, intExt, location, timeOfDay, pages, synopsis, characters[], props[], costumes[], makeupFx[], notes |
| `casting.ts` | Casting calls + confirmed cast members |
| `crew.ts` | Crew members grouped by department |
| `shots.ts` | Shot list — shotType, cameraMovement, cameraBody, lens, description, status |
| `budget.ts` | Budget line items — category, estimatedAmount, actualAmount |
| `lighting.ts` | Lighting diagrams (canvas state saved as JSON) |
| `packets.ts` | Production packets |
| `post_production.ts` | Post milestones + deliverables |
| `distribution.ts` | Distribution submissions |
| `distribution_strategy.ts` | Press kit / strategy — one record per project |
| `tools.ts` | Production toolchain |

**Scene array fields:** `characters`, `props`, `costumes`, `makeupFx` stored as `text` in DB, typed as `string[]` in OpenAPI. Always: `Array.isArray(scene.characters) ? scene.characters : []`

### 15 Modules
1. Projects List
2. Dashboard (4 phases, stat cards, budget progress, recent activity)
3. Development (Mind Map + Beat Sheet tabs)
4. Character Bible
5. Breakdown (Scene Reader + AD Breakdown Sheet)
6. Casting (Casting Calls + Confirmed Cast)
7. Crew (by department)
8. Shot List
9. Budget (estimated vs actual, EUR)
10. Lighting Planner (bird's-eye canvas)

12. Post-Production (milestones + deliverables)
13. Distribution (submissions tracker + press kit)
14. Tools (production toolchain registry)
15. Export (single print-ready document — footer: "Studio di Gratia — FilmCraft — [date]")

### Navigation Order
```
Dashboard → Development → Characters → Breakdown → Casting → Crew →
Shot List → Lighting → Budget → Packets → Post-Production → Distribution →
Tools → Export
```

### New Module Checklist
1. Add `lib/db/src/schema/<domain>.ts`, export from index, run push
2. Add paths + schemas to `openapi.yaml`
3. Run codegen
4. Create route file using `insertXSchema` from `@workspace/db`, wire into `routes/index.ts`
5. Create page using generated hooks from `@workspace/api-client-react`
